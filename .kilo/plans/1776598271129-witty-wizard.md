# Plan for Implementing Categorized Dropdown

## Problem
User has many expense categories (up to 20) making it difficult to select from a long dropdown when inputting transactions.

## Solution
Implement a Categorized Dropdown that groups expense categories into three main groups:
- Kebutuhan (Needs)
- Keinginan (Wants)  
- Tabungan (Savings)

This grouping will be applied to:
1. Category management interface (Settings page) - for creating/editing categories
2. Transaction input interface (Tambah/Edit Transaksi) - for selecting categories when recording transactions

Income categories and dashboard will remain unchanged.

## Implementation Steps

### 1. Update Default Categories (defaults.js) - COMPLETED
Added `categoryGroup` field to each expense category object with one of three values:
- 'kebutuhan' for essential expenses
- 'keinginan' for discretionary expenses  
- 'tabungan' for savings/investment expenses

### 2. Update AppContext (AppContext.jsx) - COMPLETED
Modified category CRUD operations to handle the new `categoryGroup` field:
- Ensured `addCategory` and `updateCategory` preserve the categoryGroup field
- Maintained backward compatibility with existing categories that may not have categoryGroup
- Updated the categories state initialization to handle migration of existing categories without categoryGroup

### 3. Update Category Settings UI (Settings.jsx -> CategoriesSettings) - COMPLETED
Added a new dropdown in the category form for selecting categoryGroup:
- Only show this dropdown when category type is 'expense'
- Options: Kebutuhan, Keinginan, Tabungan
- Map selected values to 'kebutuhan', 'keinginan', 'tabungan'
- Handle both add and edit category forms

### 4. Update Transaction Input UI (TransactionForm.jsx) - COMPLETED
Added categorized dropdown for expense categories:
- Shows category group dropdown when type is 'expense'
- Groups categories by their categoryGroup in the main category dropdown
- Preserves existing functionality for income categories
- Handles edit and prefill scenarios correctly

### 5. Update Supabase Schema - COMPLETED
Added category_group column to categories table:
```sql
ALTER TABLE categories ADD COLUMN IF NOT EXISTS category_group TEXT 
  CHECK (category_group IN ('kebutuhan', 'keinginan', 'tabungan'));

CREATE INDEX IF NOT EXISTS idx_categories_category_group ON categories(category_group);
```

### 6. Verify No Impact on Other Areas - COMPLETED
- Dashboard: Continues to work as it uses category IDs/names only
- Transaction Input: Now includes categorized dropdown for better UX
- Existing functionality: All CRUD operations work with the new field
